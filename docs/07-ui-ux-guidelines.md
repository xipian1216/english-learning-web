# UI/UX 指南

## 视觉方向

当前界面采用温暖的英语学习笔记本风格：

- 主按钮和头部区域使用橙色渐变。
- 使用圆角卡片和输入框。
- 使用柔和阴影。
- 使用浅暖色背景。
- 认证面板保持紧凑。

## 布局

### 认证布局

`AuthLayout` 将内容在视口中水平和垂直居中。登录、注册和密码重置页面使用该布局。

### 主布局

`MainLayout` 渲染 `Header`，然后渲染受保护页面内容。词汇本路由使用该布局。

## 表单

当前认证表单使用：

- 语义化 `form` 元素。
- 通过 `.sr-only` 视觉隐藏 label。
- 原生 `required` 属性。
- 浏览器 autocomplete 属性。
- 渐变主按钮。

后续表单工作应优先使用行内校验消息、加载状态和错误状态，而不是阻塞式 `alert`。

## Header

`Header` 包含：

- 来自 `/favicon.ico` 的 Logo。
- 产品提示文字：`English Learning`。
- 页面标题：`Vocabulary Notebook`。
- 面向未来搜索功能的输入框占位符。

当前搜索输入框仅用于视觉展示，不会过滤词汇本内容。

## 响应式行为

- `Header` 在宽度低于 `720px` 时纵向排列。
- `VocabularyNotebook` 在宽度低于 `768px` 时切换为单列布局。
- 认证表单使用 `max-width: 92vw` 限制宽度。

## 可访问性要求

- 保持 label 与输入控件关联。
- 为交互元素保留可见名称或程序可读名称。
- 为自定义输入框和按钮保留焦点样式。
- 按页面层级使用语义化标题。
- 错误状态不要只依赖颜色表达。

## 随功能成熟需要定义的状态

- 认证 API 调用加载状态。
- 行内校验错误。
- 空词汇本状态。
- API 或网络错误状态。
- 创建、更新、删除成功反馈。
- 防重复提交的禁用状态。
